# 九州南北朝ヘックス戦記 プロジェクトメモ (GEMINI.md)

このファイルは、プロジェクトの文脈と継続的な開発指針を維持するためのものです。

## 1. プロジェクトの目的
南北朝時代の九州（征西府）をテーマにした、ブラウザ（Tauri）で動作するヘックスベースの戦略シミュレーションゲームを構築する。

## 2. コンテキスト
- **時代背景:** 日本の南北朝時代（14世紀）、懐良親王と菊池武光による征西府の全盛期から、今川了俊による北朝の反撃まで。
- **主要人物:** 懐良親王（南朝）、菊池武光（南朝）、今川了俊（北朝）、一色範氏（北朝）、少弐頼尚（中立/転向）。
- **主要地点:** 大宰府、肥後・菊池、博多。

## 3. 技術スタック
- **Frontend:** React + TypeScript (Vite)
- **Backend:** Rust (Tauri v2)
- **Drawing:** HTML5 Canvas API (External libraries not used for hex rendering)
- **Data Strategy:** Rust側で `get_game_data` コマンドを通じて歴史データを提供。

## 5. 次回の課題・実装案
- **AIの実装:** 北朝ターンでの自動移動・攻撃ロジックの高度化（現在は簡易的な `useAI` を使用）。

- **勝利条件:** 特定のユニット（懐良親王や今川了俊）の撃破による勝敗判定。
- **UI改善:** 地形情報の詳細表示（ツールチップなど）。
- **移動範囲の可視化:** 選択中のユニットが移動可能なヘックスをハイライトする。

## 6. 開発ガイドライン
- コード変更後は、必ず `kyushu_nanbokucho_data.json`（または埋め込まれたJSON）の整合性を保つこと。
- ヘックス計算には Axial Coordinate 方式を使用し、`cubeRound` を介した変換を維持すること。
- 九州の地理的特徴（海、山、城）をヘックスグリッドのメタデータとして追加していくこと。
